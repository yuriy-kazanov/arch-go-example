# Architecture Rules Check

В этом репозитории используется **arch-go** для автоматической проверки архитектурных правил и зависимостей между пакетами.

Проверка помогает:

* контролировать границы слоёв и модулей
* предотвращать нежелательные зависимости
* поддерживать чистую и предсказуемую архитектуру

---

## Требования

* **Go** версии `>= 1.20`
* Установленный `git`

---

## Установка и запуск

### 1. Скачать проект

```bash
git clone https://github.com/yuriy-kazanov/arch-go-example.git
cd arch-go-example
```

---

### 2. Установить arch-go

Установка производится через `go install`:

```bash
go install github.com/fdaines/arch-go@latest
```

Убедитесь, что `$GOPATH/bin` добавлен в переменную окружения `PATH`:

```bash
export PATH="$PATH:$(go env GOPATH)/bin"
```

Проверьте, что установка прошла успешно:

```bash
arch-go --version
```

---

### 3. Запустить проверку архитектуры

В корне проекта выполните команду:

```bash
arch-go
```

* `arch-go.yml` — конфигурационный файл с архитектурными правилами
* при нарушении правил команда завершится с **ненулевым кодом выхода**

---

## Дополнительно

* Конфигурация правил находится в файле `arch-go.yml`
